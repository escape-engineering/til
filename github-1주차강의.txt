버전관리 : 프로젝트가 언제 무엇이 어떻게 바뀌었는지 알고, 관리하는 것
git 초기화 : 기존의 프로젝트를 git 프로젝트로 바꾼다!
add / staging : 스테이지에 원하는 git 프로젝트를 올린다!
commit : 언제, 누가, 변경한 점, 커밋ID 등의 Staging된 프로젝트 정보를 특정 제목과 함께 저장
commit history : 프로젝트가 언제, 누가, 어떤걸 바꾸었는지 지정한 commit 내용을 제목삼아 한 눈에 보여준다
tracking :  로컬 repo와 원격 repo를 연결한다!
pull : 원격 repo에서 특정 프로젝트를 로컬로 가져오거나 최신화 시킨다.
push : 로컬 repo의 프로젝트를 원격 repo에 새로 업데이트하거나 새롭게 업로드 합니다.
clone : 누군가 만들어둔 원격 repo의 git project를 처음으로 다운로드한다!
